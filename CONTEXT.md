# Todo Block

本应用围绕按层级组织待办，并让用户以符合直觉的方式改变和恢复这些待办。

## Language

**父子组**：
一项待办与它下面归属于它的全部后代组成的不可拆分单位。移动目标待办时始终带上整个父子组，待办是否完成不改变父子归属。
_Avoid_: 单行、已选项目、已完成项目

**操作单元**：
真正改变用户状态的一次明确意图及其引发的全部连锁变化，包括相关的选择与编辑焦点。一个操作单元对应一次完整撤销，也对应一次完整恢复；没有造成变化的动作不构成操作单元。
_Avoid_: 单次变更、撤销记录、事务

**连续输入段**：
用户在同一待办中连续进行同一种文字编辑，直到停顿一秒、切换编辑方式、移动光标、改变选区、切换待办或执行其他操作。连续输入、连续删除和选区替换分别成段；输入法的一次完整选词属于同一段。一次语音输入从开始到明确结束也属于同一段，期间的停顿和对既有识别文字的动态改写不会拆段。用户在临时文字尚未确认时执行其他明确动作，应用会先结束并保存完整输入段，再执行下一动作。撤销与恢复同时还原该段前后的文字光标和文字选区。
_Avoid_: 单次按键、整场编辑

**用户状态**：
用户能直接感知的待办内容、完成状态、层级、归属、顺序、选择与编辑焦点。撤销和恢复必须严格还原用户状态，但无需倒回仅供后台使用的记录。
_Avoid_: 完整内部状态、存储状态

**纯导航**：
只改变当前选择、编辑焦点或光标位置，而不改变待办内容与组织方式的行为。纯导航不进入操作历史，也不结束已有的恢复路径。
_Avoid_: 编辑操作、空操作

**可见自动操作**：
由应用发起、会明显改变用户状态的行为，例如昨日结转。它与手动操作一样进入操作历史，并能完整撤销和恢复。
_Avoid_: 后台整理、维护任务

**待办身份**：
一项待办在创建、删除、撤销和恢复过程中保持不变的自身身份。撤销删除与恢复新建时回来的是原来的待办，而不是内容相同的新待办。
_Avoid_: 待办副本、重建待办

**完整应用**：
一个操作单元的全部变化共同成功或共同不生效。撤销和恢复不得留下部分改变的父子组或用户状态。
_Avoid_: 尽力恢复、部分成功

**操作历史**：
当前运行期间全应用共用、按发生顺序排列的操作单元序列，且只有一条前进方向；文字编辑与结构操作使用同一历史。撤销后发生新操作时，尚未恢复的旧操作立即退出历史；应用重启后开始新的操作历史。菜单栏不会建立或筛选另一份历史，但只有在历史下一步会直接改变今天列表时才能执行撤销或恢复；它不能越过不可见步骤处理更早的今天操作。
_Avoid_: 页面历史、列表历史、窗口历史

**当前列表**：
此刻接收应用级命令的唯一待办列表。列表中的直接用户动作会先令该列表接管，再执行动作；菜单栏面板打开期间由菜单栏列表立即接管，无需先点击其中的待办。延迟保存、输入段归档和界面刷新不会引发接管。临时接管结束后恢复仍然存在的原列表；原列表已消失时保持没有当前列表，直到另一列表主动接管。各列表独立保留自己的选择与文字光标，接管不会覆盖其他列表保留的状态。
_Avoid_: 所有可见列表、最近出现的列表

**菜单栏列表**：
只展示并操作今天待办的快速入口，支持今天待办的新增、删除、完成、文字和层级修改，但不负责把待办拖到未来月份或长期列表。
_Avoid_: 主窗口缩略版、跨列表入口

**文字编辑状态**：
文字光标或文字选区仍位于待办标题中的状态。此时复制、剪切、粘贴和全选优先作用于标题文字；只有文字光标但没有文字选区时，复制与剪切不可用，不会转而处理整条待办。退出该状态后，这些命令才作用于整条待办或多条待办。
_Avoid_: 待办选择、当前列表

**未执行动作**：
用户表达了明确意图，但动作因当前边界或临时条件而没有改变用户状态。可见边界安静处理；用户无法看见的原因提供不打断输入的短暂说明；只有不可逆风险才要求用户确认。
_Avoid_: 失败操作、空操作、程序错误
