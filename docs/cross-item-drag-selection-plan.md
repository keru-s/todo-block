# 跨项拖选方案

> 状态：已完成，已验证。

## 目标

用户从待办标题文字或行内空白开始拖动时，既能正常编辑和选择文字，也能在鼠标越过当前待办的空间边界后，连续选择同一分组中的多条待办。这个行为不依赖按住时间，也不替代左侧把手原有的待办移动能力。

## 已确认的行为

### 从哪里开始

- 从标题文字开始：先按文字选择处理。
- 从待办行的空白处开始：先让被点击的待办进入文字编辑，光标落在离点击位置最近的文字位置；拖动是否进入待办选择，仍由下方的空间边界决定。
- 从完成方框开始：只改变完成状态。
- 从左侧拖动把手开始：只移动待办。

### 文字选择与待办选择的边界

- 不使用按住时间判断意图。
- 上下方向以起始待办整行的上边和下边为界；鼠标没有越过当前待办时，继续按文字选择处理，越过后进入待办选择。
- 左右方向以标题文字框为基础，左右各增加三个汉字宽的保护区；鼠标没有越过保护区时，继续按文字选择处理，越过后进入待办选择。
- 斜向移动仍优先选择文字，直到鼠标越过上述任一空间边界。
- 进入待办选择时，文字选区和文字光标立即消失，只显示整条待办的选中状态。
- 一旦进入待办选择，本次手势不再恢复成文字选择。

### 选择范围

- 每次拖选都以鼠标按下的待办为新起点，并替换拖选前的选择。
- Shift、Command 等按键不追加、排除或切换拖选结果；跨项拖选始终建立一段新的连续选择。原有的 Shift 点击多选保持不变。
- 反向拖动时，经过的待办逐条退出选择。
- 回到起始待办后仍保持整条待办选中；继续越过起始待办，可以改为选择另一方向的连续待办。
- 两条待办之间以空隙中线为边界。越过中线，下一条才加入选择；反向越回中线，下一条退出选择。
- 日期列表只在起始待办当天内选择，不跨天。
- 长期待办只在起始待办所属的“紧急”或“重要”分组内选择，不跨分组。

### 与完成和移动操作的关系

- 多选状态下，拖动已选待办的左侧把手，会移动全部已选待办；父待办仍按既有规则带上全部子待办。
- 拖动未选待办的把手，会结束原多选并只移动该待办；只单击把手而没有拖动时，不改变选择。

### 滚动与结束

- 鼠标停留在可见区域顶部或底部附近时，列表自动滚动并继续扩展选择。
- 自动滚动到起始分组边界后停止。
- 鼠标按住并从窗口上方或下方移出时，继续自动滚动；在窗口外松开时，以当时的选择作为最终结果。
- 松开鼠标后保持整条待办的选择状态，不自动返回文字编辑。
- 即使最终只选中起始待办一条，也保持整条待办选择；再次点击标题才返回文字编辑。
- 拖选过程中按下 Escape，会取消本次拖选，并恢复拖选开始前的待办选择和文字状态。
- 窗口失去使用状态时结束当前手势并保留当时的选择；窗口再次使用时，选择仍然存在，直到用户普通单击。
- 多选后单击某条待办的标题，会结束原多选，只保留被点击的待办，并把文字光标放到点击位置进入编辑。
- 单击待办行内部、但不在文字上的空白，会清除全部整条待办选择，并让被点击的待办进入文字编辑。光标落在离点击位置最近的文字位置：标题左侧对应标题开头，右侧对应最近一行末尾，多行标题按点击高度选择最近的一行。
- 单击所有待办之外的列表背景，会清除全部选择并退出文字编辑。
- 多选状态下，单击选中范围内任一待办的完成方框，会对全部已选待办执行同一次完成状态变更，并保留当前选择。点击未完成待办的空方框时全部设为完成；点击已完成待办的勾选方框时全部设为未完成。
- 多选状态下，单击未选中待办的完成方框，只切换被点击待办的完成状态，原有多选范围保持不变。
