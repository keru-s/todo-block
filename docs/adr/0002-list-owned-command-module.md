# 由当前列表拥有统一动作模块

每个可见待办列表独立拥有一份动作模块，列表内操作与应用级命令都通过它进入；当前列表接管应用级命令，临时界面关闭后恢复仍然存在的原列表。模块统一归属、可用判断、执行结果和反馈，但保留两类清晰表达：菜单与快捷键使用统一命令，文字、拖拽等编辑器动作使用各自的专用形式。窗口、页面导航、菜单栏面板和应用设置仍由外层界面负责。撤销与恢复仍使用全应用唯一的操作历史，当前列表只接管命令和结果展示，不按列表切割历史；展示范围受限的菜单栏只有在历史下一步会直接改变今天列表时才能执行，且不能跳过其他步骤。这样可以删除多个全局入口各自保存当前列表的分裂状态，同时避免把差异很大的动作强塞进一个巨型通用命令或整个应用的控制中心。

新旧入口不在产品中长期并存。迁移可以分步开发，但三个列表入口全部切换并通过验收后立即删除旧动作工厂和各自保存当前列表的管理器；需要回退时使用版本历史。

## Considered Options

- 保留剪贴板、移动和历史展示各自的接管入口：容易让同一时刻的命令指向不同列表。
- 将所有编辑器动作包装为同一种通用命令：表面统一，但会让文字输入、语音输入和拖拽携带大量互不相干的信息。
