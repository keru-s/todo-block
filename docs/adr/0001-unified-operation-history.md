# 用统一操作单元管理待办变更与撤销

所有会改变待办的行为都必须经过同一个变更入口，并以最外层用户意图形成一个操作单元。统一入口在完整检查后一次应用全部变化，保存变化前后的用户状态，并把操作加入全应用共用的单线历史；内部的新建、移动、删除、层级调整和自动整理只贡献变化，不各自形成历史。这样选择集中管理而不是由各功能分别修改和补写撤销，是为了保证批量、复合及自动操作始终完整成功或完整不生效，并让撤销与恢复严格对称。

操作历史只属于当前运行，最多保留最近 50 个完整操作单元。纯导航、复制、无结果动作和不可见的后台工作不进入历史；文字编辑按连续输入段进入同一条全局历史。撤销后发生新的可见变化会结束旧的恢复路径，可见自动操作在恢复路径存在时不得静默执行。

## Considered Options

- **各功能分别修改并登记撤销**：局部直接，但复合操作容易被拆成多步，也容易遗漏连锁变化。
- **每步保存整个待办库副本**：恢复简单，但重复保存大量无关内容，并可能覆盖不相关状态。
- **统一入口记录完整变化集合**：只记录本次新建、删除、修改和移动，以及相关的选择与焦点；采用此方案。

## Consequences

- 一个操作单元只由最外层意图提交。批量命令、昨日结转、粘贴、拆分和自动整理都只能产生一步历史。
- 普通修改记录发生变化的内容及其前后值；新建和删除保留相关待办的完整资料，使其身份在撤销和恢复间不变。恢复使用已记录的结果，不重新运行原操作规则。
- 每一步必须完整成功或完整不生效。失效记录整条丢弃，撤销或恢复继续寻找最近一条可完整执行的记录。
- 文字即时显示并保存。连续输入、连续删除和选区替换分别成段，以一秒停顿或光标、选区、焦点及操作变化为边界；输入法未确认内容先由输入法处理。
- 纯导航和剪贴板变化不进入历史，也不结束恢复路径。撤销剪切只恢复待办或文字，不改写系统剪贴板。
- 撤销与恢复还原相关选择、焦点、文字光标和文字选区，并切换到能显示结果的入口；只要求目标可见，不恢复精确滚动位置。
- 菜单保持简单的“撤销”和“恢复”，没有可执行记录时置灰，不要求展示具体操作名称。
