# Domain docs

This repository uses a single domain context.

## Before working

- Read `CONTEXT.md` at the repository root when it exists.
- Read relevant decisions under `docs/adr/` when they exist.
- Missing files are not an error; continue without asking for them to be created.

## Vocabulary

Use terms defined in `CONTEXT.md` consistently in plans, issue titles, tests, and design proposals. If an important concept is missing, record it when the concept has been settled.

## Existing decisions

If new work conflicts with a recorded decision, point out the conflict instead of silently replacing the earlier decision.
