# Issue tracker: GitHub

Issues and specs for this repository live in GitHub Issues. Use the `gh` command-line tool and infer the repository from the configured Git remote.

## Common operations

- Create, read, list, comment on, label, and close issues through `gh issue`.
- When another engineering skill asks to publish a spec or ticket, create a GitHub issue.
- When it asks to fetch a ticket, read the issue together with its comments and labels.
- Pull requests are not treated as incoming requests for triage.

## Large-work planning

A large plan is represented by one map issue and several linked child issues. Blocking relationships should use GitHub’s built-in issue dependencies when available, with a written fallback when unavailable.
